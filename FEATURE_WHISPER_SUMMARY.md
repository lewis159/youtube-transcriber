# Feature: Whisper Transcription + AI Summary

Branch: `feature/whisper-summary` · Worktree: `C:\dev\yt-whisper-summary` · Status: built, **fully dark by default**, never merged to master.

---

## 1. Overview

This feature adds two capabilities to the YouTube Transcriber app, both **fully additive and flag-gated** (default OFF for every tier):

- **Local Whisper transcription fallback** — an in-house `faster-whisper` worker that transcribes videos when the YouTube caption scrape is unavailable/insufficient. Uploads with the `stt_fallback` flag ON are enqueued to a Redis/BullMQ `transcription` queue and processed asynchronously by the worker; with the flag OFF the existing synchronous caption-scrape path runs unchanged.
- **Claude per-tier AI summaries + summary page** — on-demand summary, key points and chapters generated by Claude (model chosen per tier), surfaced on a dedicated summary page with a seekable embedded player, clickable chapters, a searchable full transcript, a branded summary-PDF export, and a Q&A chat that answers questions from the transcript.

Three new flags gate the whole surface: `stt_fallback`, `ai_summary`, `summary_chat`. They are seeded `enabled = false` for all tiers in migration `010`, so nothing changes until a tier is explicitly switched on. Every new user-facing surface verifies its flag before doing anything (see the gating audit summary below). Nothing was merged to master; the entire feature lives on this branch/worktree.

## 2. Components & files

**Worker (separate, separately deployable):**

- `whisper-worker/Dockerfile`, `requirements.txt` — container build + Python deps.
- `whisper-worker/benchmark.py`, `BENCHMARK_README.md` — model/throughput benchmark harness (follow-up #7).

**App additions (`app-ha/`):**

- `app/api/videos/upload/route.ts` — adds the `stt_fallback`-gated async enqueue branch; existing sync caption path is the `else`.
- `lib/claude.ts` — Claude summarisation layer: per-tier model map, transcript cap, structured summary builder.
- `app/api/videos/[id]/summary/route.ts` — `ai_summary`-gated GET (cached summary) + POST (generate-and-cache, idempotent).
- `app/api/videos/[id]/chat/route.ts` — `summary_chat`-gated streaming Q&A over the transcript.
- `lib/youtube-seek.ts` — postMessage helper to seek the embedded player (chapters/timestamps).
- `app/(auth)/video/[id]/summary/page.tsx` — server component: auth, ownership, resolves flags, renders the view.
- `app/(auth)/video/[id]/summary/SummaryView.tsx` — client: player + summary + chapters + searchable transcript + summary-PDF button; all gated bits inert when OFF.
- `app/(auth)/video/[id]/summary/SummaryChat.tsx` — client streaming chat panel (only mounted when chat enabled).
- `lib/export.ts` — adds `generateSummaryPDF(...)` (branded summary PDF).
- `app/api/videos/[id]/export/route.ts` — adds the `ai_summary`-gated `format=summary-pdf` branch.
- `app/(auth)/video/[id]/page.tsx` — adds the "AI Summary" link (shown when status completed).

## 3. How to turn it ON

1. **Run the migration** — apply `supabase/migrations/010_whisper_summary.sql` to the project. Additive and idempotent; safe on a live DB.
2. **Deploy the worker** — bring up Redis first, then the worker:
   ```bash
   docker compose -f docker-compose.redis.yml up -d            # existing Redis (if not already up)
   docker compose -f docker-compose.whisper.yml up -d --build  # the new worker
   ```
   The worker shares the Xeon box with Ollama for now, which is why CPU is capped (~2 cores / 4G, `WHISPER_CPU_THREADS=2`). Raise the caps and thread count once it moves to dedicated hardware.
3. **Set env** (worker container + app — see §4): `ANTHROPIC_API_KEY`, `WHISPER_MODEL`, `WHISPER_CPU_THREADS`, Redis vars (`REDIS_HOST`/`REDIS_PORT` or `REDIS_URL`), Supabase vars (`SUPABASE_URL`, `SUPABASE_SERVICE_KEY` / `SUPABASE_SERVICE_ROLE_KEY`).
4. **Enable the flags per tier** — in the `tier_features` table (or via the admin feature-flags page), set `enabled = true` for the tiers that should get each of:
   - `stt_fallback` — routes uploads through the Whisper worker.
   - `ai_summary` — summary page, summary GET/POST, summary-PDF export.
   - `summary_chat` — Q&A chat.

   Enabling can be done independently and is instant (no deploy). With all flags OFF the app behaves exactly as before.

## 5. CLEAN TEARDOWN checklist

Do these in order for zero impact on the main app:

1. **Flip the three flags OFF** — set `enabled = false` for `stt_fallback`, `ai_summary`, `summary_chat` in `tier_features` (or the admin page). Instant neutralise, no deploy: every new surface goes inert immediately.
2. **Stop/remove the worker:**
   ```bash
   docker compose -f docker-compose.whisper.yml down -v --rmi local
   ```
   (Only touches the `whisper-worker` service, its named volume, and its locally built image. Leaves Redis and all pre-existing containers untouched.)
3. **Run the DB rollback** — apply `supabase/migrations/010_whisper_summary_down.sql` (drops the added `transcripts` columns, the `transcription_jobs` + `video_summaries` tables, and the 3 seeded flags).
4. **Remove the files** (and revert the additive branches):
   - Delete: `whisper-worker/`, `docker-compose.whisper.yml`, `app-ha/lib/claude.ts`, `app-ha/lib/transcription-queue.ts`, `app-ha/lib/youtube-seek.ts`, `app-ha/app/api/videos/[id]/summary/`, `app-ha/app/api/videos/[id]/chat/`, `app-ha/app/(auth)/video/[id]/summary/`.
   - Revert the additive edits in: `app-ha/app/api/videos/upload/route.ts` (drop the `stt_fallback` enqueue branch), `app-ha/app/api/videos/[id]/export/route.ts` (drop the `summary-pdf` branch), `app-ha/lib/export.ts` (drop `generateSummaryPDF`), `app-ha/app/(auth)/video/[id]/page.tsx` (drop the "AI Summary" link), `app-ha/app/(auth)/dashboard/page.tsx` (revert the status-set additions).
   - Drop `bullmq` and `@anthropic-ai/sdk` from `app-ha/package.json` — verified nothing else in the app uses them (`bullmq` only in `transcription-queue.ts`; `@anthropic-ai/sdk` only in `claude.ts` and the chat route).
5. **Or simplest of all** — since nothing was ever merged to master, drop the whole branch:
   ```bash
   git worktree remove C:/dev/yt-whisper-summary
   git branch -D feature/whisper-summary
   ```
   (Still run steps 1–3 against any environment where the migration/worker were actually deployed — the worktree removal only removes the code.)

## 6. Known follow-ups

- **Worker is not Sentinel-aware** — it connects to `redis-master` directly (not via the global socket-proxy / HA discovery). Revisit when the queue moves under Sentinel's job-queue worker pool.
- **Benchmark (#7) still needs running on the box** — to set the real default `WHISPER_MODEL`; current default is `base` as a placeholder. Use `whisper-worker/benchmark.py`.
- **Cost quotas are stubs** — `transcription_jobs.cost_usd` and the summary transcript cap exist, but there is no per-user/per-tier spend enforcement yet. The transcript char cap (`MAX_TRANSCRIPT_CHARS = 120k`) is the only active guardrail.
